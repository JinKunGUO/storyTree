## 数据类型分析

### 1. 章节文本内容

| 特征 | 说明 |
|------|------|
| **数据量** | 单章节 1KB-50KB，大型小说可达数百 MB |
| **访问模式** | 读多写少，按需加载 |
| **一致性要求** | 高（不能丢失用户创作） |
| **查询需求** | 需要全文搜索、按故事/章节查询 |

### 2. 图片文件

| 特征 | 说明 |
|------|------|
| **数据量** | 单文件 100KB-5MB |
| **访问模式** | 读多写少，需要 CDN 加速 |
| **一致性要求** | 中等 |
| **查询需求** | 仅按 URL 访问 |

### 3. 元数据

| 特征 | 说明 |
|------|------|
| **数据量** | 较小（用户信息、故事信息等） |
| **访问模式** | 读写频繁 |
| **一致性要求** | 高（涉及积分、会员等） |
| **查询需求** | 复杂关联查询 |

```

│                   云主机                      │
│  ┌─────────────┐  ┌─────────────────────┐   │

│  └─────────────┘  └─────────────────────┘   │

│  │         Node.js 应用                 │    │
│  └─────────────────────────────────────┘    │
```

| 优点 | 缺点 |
|------|------|
| ✅ 部署简单 | ❌ 单点故障风险 |

**适用场景**：开发测试、小规模部署（< 10GB 数据）

## 推荐架构方案

### 核心原则

| 数据类型 | 推荐存储 | 原因 |
|----------|----------|------|
| **章节文本** | 云数据库（MySQL/PostgreSQL） | 需要事务、全文搜索、关联查询 |
| **图片文件** | OSS + CDN | 静态资源，需要加速，节省带宽 |
| **热点数据** | Redis | 缓存热门章节、用户会话、频率限制 |
| **元数据** | 云数据库 | 需要 ACID 事务保证 |

### 为什么章节文本不放 OSS？

| 考量 | OSS | 数据库 |
|------|-----|--------|
| **全文搜索** | ❌ 不支持 | ✅ 原生支持 |
| **事务一致性** | ❌ 无 | ✅ ACID |
| **关联查询** | ❌ 不支持 | ✅ JOIN 查询 |
| **版本控制** | ⚠️ 需额外实现 | ✅ 可直接实现 |
| **批量更新** | ❌ 困难 | ✅ 简单 |

**结论**：章节文本是结构化数据，应存储在数据库中，而非 OSS。

### 为什么图片放 OSS？

| 考量 | 本地磁盘 | OSS |
|------|----------|-----|
| **CDN 加速** | ❌ 需额外配置 | ✅ 原生支持 |
| **存储成本** | 💰 云主机磁盘贵 | 💵 OSS 便宜 |
| **扩展性** | ❌ 受限于单机 | ✅ 无限扩展 |
| **备份** | ❌ 需手动 | ✅ 自动冗余 |
| **带宽** | ❌ 占用主机带宽 | ✅ 独立带宽 |

**结论**：图片应存储在 OSS，通过 CDN 分发。

### Redis 的作用

| 用途 | 说明 |
|------|------|
| **热点章节缓存** | 缓存阅读量高的章节，减少数据库压力 |
| **用户会话** | 存储登录状态、token |
| **频率限制** | API 限流、防刷 |
| **AI 任务队列** | 异步任务处理 |
| **排行榜** | 热门故事、活跃用户排行 |

**注意**：Redis 是缓存层，不应作为持久化存储。

## 迁移建议

### 1. 数据库迁移（SQLite → MySQL）

```bash
# 1. 修改 prisma/schema.prisma
datasource db {
  provider = "mysql"  # 改为 mysql
  url      = env("DATABASE_URL")
}

# 2. 更新 .env
DATABASE_URL="mysql://user:password@host:3306/storytree"

# 3. 重新生成 Prisma Client
npx prisma generate

# 4. 数据迁移
npx prisma migrate deploy
```

### 2. 图片迁移（本地 → OSS）

**修改 `api/src/utils/upload.ts`**：

```typescript
import OSS from 'ali-oss';

const ossClient = new OSS({
  region: process.env.OSS_REGION,
  accessKeyId: process.env.OSS_ACCESS_KEY_ID,
  accessKeySecret: process.env.OSS_ACCESS_KEY_SECRET,
  bucket: process.env.OSS_BUCKET,
});

export const uploadToOSS = async (file: Buffer, filename: string): Promise<string> => {
  const result = await ossClient.put(`uploads/${filename}`, file);
  return result.url;
};
```

### 3. 添加 Redis 缓存

**热点章节缓存示例**：

```typescript
import Redis from 'ioredis';

const redis = new Redis(process.env.REDIS_URL);

// 获取章节（带缓存）
async function getNode(nodeId: number) {
  const cacheKey = `node:${nodeId}`;
  
  // 先查缓存
  const cached = await redis.get(cacheKey);
  if (cached) {
    return JSON.parse(cached);
  }
  
  // 缓存未命中，查数据库
  const node = await prisma.nodes.findUnique({ where: { id: nodeId } });
  
  // 写入缓存（1小时过期）
  await redis.setex(cacheKey, 3600, JSON.stringify(node));
  
  return node;
}
```

## 总结

### 回答你的问题

| 数据类型 | 应该存哪里 | 原因 |
|----------|------------|------|
| **公版书/用户创作文本** | **云数据库** | 需要全文搜索、事务、关联查询 |
| **图片（封面/头像）** | **OSS + CDN** | 静态资源，需要加速，节省带宽 |
| **热点数据缓存** | **Redis** | 提升访问速度，减少数据库压力 |

### 不推荐的方案

| 方案 | 问题 |
|------|------|
| ❌ 章节文本存 OSS | 无法全文搜索、无事务、查询困难 |
| ❌ 章节文本存 Redis | Redis 是内存数据库，成本高、数据量受限 |
| ❌ 图片存数据库 | 浪费数据库资源、无法 CDN 加速 |

### 推荐的演进路径

```
当前（开发）          →  阶段一（小规模）      →  阶段二（中规模）
SQLite + 本地磁盘     →  MySQL + OSS + Redis  →  读写分离 + ES
```
