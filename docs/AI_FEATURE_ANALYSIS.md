## ❌ 存在的问题

### 🔴 严重问题

#### 问题1: API参数不匹配
**描述**: 前端调用AI API时传入的参数与后端期望的参数不一致

**前端代码**:
```javascript
// web/write.html line 991
body: JSON.stringify({
    storyId: parseInt(storyId),  // ❌ 前端传storyId
    context: currentContent,      // ❌ 前端传context
    count: 3
})
```

**后端代码**:
```javascript
// api/src/routes/ai.ts line 26
const { nodeId, style, count = 3 } = req.body;  // ❌ 后端期望nodeId

if (!nodeId) {
    return res.status(400).json({ error: 'nodeId is required' });
}
```

**影响**: AI功能完全无法使用，用户点击AI续写按钮后会收到"nodeId is required"错误

**解决方案**: 
- 方案A: 修改后端支持storyId和context参数
- 方案B: 修改前端传入nodeId参数
- **推荐方案A**: 更灵活，支持未保存章节的AI续写

---

#### 问题2: 写作页面无nodeId
**描述**: 用户在write.html页面首次写作时，还没有创建节点，因此没有nodeId

**场景**:
1. 用户创建新故事
2. 点击"开始写作"
3. 写了一些内容想要AI续写
4. ❌ 此时没有nodeId，无法调用AI API

**影响**: 新用户首次写作时无法使用AI功能

**解决方案**:
- 方案A: 先创建草稿节点，再调用AI
- 方案B: 修改AI API支持基于story和context生成
- **推荐方案B**: 用户体验更好

---

#### 问题3: 认证Token处理不一致
**描述**: 前端AI功能的认证逻辑与其他功能不一致

**代码问题**:
```javascript
// write.html line 978
const userId = req.headers['x-user-id'];  // ❌ 使用x-user-id header
```

但其他路由使用JWT token认证。

**影响**: 可能导致认证失败

**解决方案**: 统一使用JWT middleware进行认证

---
