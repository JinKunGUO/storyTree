### 2. 插图持久化功能

#### 后端实现（`api/src/routes/nodes.ts`）

**修改内容**：
- ✅ `PUT /api/nodes/:id` 路由支持 `image` 字段更新
- ✅ 动态构建更新数据，支持可选的 `image` 字段
- ✅ 支持三种场景：
  - 应用插图：保存图片URL
  - 编辑章节：保持原有插图
  - 删除插图：设置为null

**代码改进**：
```typescript
// 提取 image 参数
const { title, content, image } = req.body;

// 动态构建更新数据
const updateData: any = {
    title,
    content,
    updated_at: new Date(),
    review_status: reviewCheck.needReview ? 'PENDING' : 'APPROVED'
};

// 如果提供了image字段，则更新它
if (image !== undefined) {
    updateData.image = image || null;
}
```

**解决的问题**：
- ❌ **修复前**：插图URL没有保存到数据库，刷新后丢失
- ✅ **修复后**：插图URL正确保存，刷新/重新加载后仍然显示

### 3. 插图显示功能

#### 前端实现（`web/chapter.html`）

**修改内容**：
- ✅ 修改 `renderChapter()` 函数，支持插图显示
- ✅ 插图在章节内容中间显示
- ✅ 计算插入位置，在段落分隔符处插入
- ✅ 响应式图片样式，适配各种屏幕

**显示逻辑**：
```javascript
if (chapter.image) {
    // 计算插入位置（内容中间）
    const insertPosition = Math.floor(contentLength / 2);
    
    // 找到附近的段落分隔符
    let actualPosition = insertPosition;
    for (let i = insertPosition; i < Math.min(insertPosition + 100, contentLength); i++) {
        if (content[i] === '\n') {
            actualPosition = i + 1;
            break;
        }
    }
    
    // 在内容中间插入插图
    chapterTextEl.innerHTML = `
        <div class="chapter-text-part">${escapeHtml(content.substring(0, actualPosition))}</div>
        <div class="chapter-illustration">
            <img src="${chapter.image}" alt="章节插图" />
        </div>
        <div class="chapter-text-part">${escapeHtml(content.substring(actualPosition))}</div>
    `;
}
```

- ✅ 修改 `applyIllustration()` 函数，优化应用插图流程
- ✅ 应用后直接更新当前页面，无需刷新
- ✅ 更新本地数据，立即重新渲染

**用户体验改进**：
- 插图显示在内容中间，视觉效果更好
- 图片响应式适配，移动端体验优化
- 应用插图后立即显示，无需等待
