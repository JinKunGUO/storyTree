# 故事权限管理系统 - 实现总结

## 已完成功能

### 1. 数据库扩展 ✅

#### stories表新增字段
- `visibility`: 可见性（public/private/friends）
- `password`: 密码保护（可选）
- `allow_branch`: 是否允许他人续写
- `allow_comment`: 是否允许评论
- `tags`: 标签（JSON数组）

#### 新增表
- **story_collaborators**: 故事协作者表
  - `story_id`: 故事ID
  - `user_id`: 用户ID
  - `role`: 角色（owner/editor/viewer）
  - `invited_by`: 邀请人ID
  - `created_at`: 创建时间

- **friendships**: 好友关系表
  - `user1_id`: 用户1 ID
  - `user2_id`: 用户2 ID
  - `status`: 状态（pending/accepted/rejected）
  - `created_at`: 创建时间

### 2. 权限检查中间件 ✅

**文件**: `api/src/utils/permissions.ts`

#### 核心函数
- `canViewStory()`: 检查查看权限
  - 公开故事：所有人可见
  - 私密故事：仅作者和协作者
  - 仅好友：作者、协作者和好友

- `canEditStory()`: 检查编辑权限
  - 作者：完全权限
  - 协作者（editor/owner）：可编辑

- `canCreateBranch()`: 检查创建分支权限
  - 根据`allow_branch`设置
  - 作者和协作者始终可以

- `canComment()`: 检查评论权限
  - 根据`allow_comment`设置
  - 需要有查看权限

#### 中间件
- `checkViewPermission`: 查看权限中间件
- `checkEditPermission`: 编辑权限中间件
- `checkBranchPermission`: 分支权限中间件

### 3. 后端API ✅

**文件**: `api/src/routes/stories.ts`

#### 权限设置
- `PUT /stories/:id/settings`: 更新故事权限设置
  - 可见性、密码、分支、评论、标签

#### 协作者管理
- `POST /stories/:id/collaborators`: 添加协作者
- `DELETE /stories/:id/collaborators/:userId`: 移除协作者
- `GET /stories/:id/collaborators`: 获取协作者列表
- `POST /stories/:id/collaborators/leave`: 退出协作

#### 故事列表
- `GET /stories/my`: 获取我的故事（创建的+协作的）
- `GET /stories/shared-with-me`: 获取分享给我的故事

#### 集成权限检查

- `GET /stories/:id/tree`: 应用查看权限检查

## 使用流程

### 作为故事作者

1. **创建故事后，设置权限**
   ```
   story.html → 点击"设置"按钮 → story-settings.html
   ```

2. **设置可见性**
   - 公开：所有人可见
   - 私密：仅自己和协作者
   - 仅好友：自己、协作者和好友

3. **添加协作者**
   - 在"协作者管理"区域搜索用户
   - 选择角色（编辑者/查看者）
   - 点击"添加"

4. **管理所有故事**
   ```
   导航栏 → 我的故事 → my-stories.html
   ```

### 作为协作者

1. **接收邀请**
   - 收到通知（协作邀请）
   - 点击通知链接查看故事

2. **查看协作故事**
   ```
   导航栏 → 我的故事 → 筛选"我协作的"
   ```

3. **退出协作**
   ```
   我的故事 → 找到协作故事 → 点击"退出协作"
   ```

### 作为读者

1. **查看公开故事**
   - 无需登录，直接访问

2. **查看仅好友故事**
   - 需要与作者成为好友
   - 登录后可见

3. **查看私密故事**
   - 仅作者和协作者可见
   - 其他人访问返回403错误
