## 🎯 二、准备工作

### 1. 注册支付宝商户账号

#### 方式一：企业商户（推荐）
1. 访问 [支付宝开放平台](https://open.alipay.com/)
2. 使用企业支付宝账号登录
3. 完成开发者认证
4. 准备材料：
   - 营业执照
   - 法人身份证
   - 企业对公账户

#### 方式二：个人商户（功能受限）
1. 个人账号暂时无法直接申请当面付
2. 可以通过第三方服务商接入
3. 或者使用"个人收款码"（不推荐，无法自动化）

### 2. 创建应用并配置

#### 步骤 1：创建应用
1. 登录 [支付宝开放平台](https://open.alipay.com/)
2. 进入"开发者中心" → "网页/移动应用"
3. 点击"创建应用"
4. 填写应用信息：
   - 应用名称：StoryTree
   - 应用图标：上传 Logo
   - 应用描述：AI 故事创作平台
   - 应用首页：https://yourdomain.com

#### 步骤 2：添加产品
在应用中添加以下产品：
- **当面付**（产品码：FACE_TO_FACE_PAYMENT）
- **手机网站支付**（产品码：WAP_WAP）

#### 步骤 3：签约产品
1. 点击"签约"按钮
2. 提交签约材料
3. 等待审核（通常 1-3 个工作日）

#### 步骤 4：配置开发信息
1. 进入"开发信息"页面
2. 生成"应用公钥"和"应用私钥"：
   ```bash
   # 使用 OpenSSL 生成 RSA 密钥对
   openssl genrsa -out app_private_key.pem 2048
   openssl rsa -in app_private_key.pem -pubout -out app_public_key.pem
   ```
3. 上传"应用公钥"到支付宝开放平台
4. 保存以下信息：
   - APP_ID
   - 应用私钥
   - 支付宝公钥（上传后系统生成）

#### 步骤 5：配置回调地址
1. 进入"开发信息" → "网关地址/回调地址"
2. 设置回调地址白名单：
   ```
   https://yourdomain.com/api/payment/alipay/notify
   ```

## ⚙️ 三、配置部署

### 1. 复制环境变量配置

```bash
cd api
cp .env.payment.example .env.payment
```

### 2. 填写配置信息

编辑 `.env.payment` 文件：

```env
# 支付宝应用 ID
ALIPAY_APP_ID=2021001234567890

# 应用私钥（去掉首尾的 PEM 标记，合并成一行）
ALIPAY_PRIVATE_KEY=MIIEvQIBADANBgkqhkiG9w0BAQEFAASC...

# 支付宝公钥（去掉首尾的 PEM 标记，合并成一行）
ALIPAY_PUBLIC_KEY=MIIBIjANBgkqhkiG9w0BAQEFAAOCAQ8A...

# 回调地址（生产环境必须是 HTTPS）
ALIPAY_CALLBACK_URL=https://api.yourdomain.com

# API URL
API_URL=https://api.yourdomain.com
```

### 3. 安装依赖

```bash
cd api
npm install alipay-sdk alipay-form
```

### 4. 环境变量合并

将 `.env.payment` 中的配置复制到 `.env` 文件中，或者在启动时加载：

```bash
# 方式一：直接复制到 .env
cat .env.payment >> .env

# 方式二：启动时指定
source .env.payment && npm run dev
```

## 🔧 五、后端 API 接口

### 1. 创建支付宝支付订单

**接口**: `POST /api/payment/alipay/create`

**请求参数**:
```json
{
  "orderId": "MEM_1234567890_ABCD"
}
```

**响应**:
```json
{
  "success": true,
  "orderId": "MEM_1234567890_ABCD",
  "qrCode": "https://qr.alipay.com/xxx",
  "amount": 39.00
}
```

### 2. 支付宝异步通知回调

**接口**: `POST /api/payment/alipay/notify`

**说明**: 支付宝服务器会向此接口发送支付结果通知

**处理逻辑**:
1. 验证签名
2. 检查订单状态
3. 更新订单为已支付
4. 发放会员权益或积分
5. 返回 "success" 或 "fail"

**幂等性保证**:
- 如果订单已支付，直接返回成功
- 避免重复发放权益

## 📊 七、订单管理

### 订单状态流转

```
pending（待支付）
  ↓
paid（已支付） → refunded（已退款）
  ↓
cancelled（已取消）
  ↓
expired（已过期）
```

### 查询订单接口

**接口**: `GET /api/payment/orders/:orderId`

**响应**:
```json
{
  "orderId": "MEM_1234567890_ABCD",
  "type": "subscription",
  "amount": 39.00,
  "status": "paid",
  "paymentMethod": "alipay",
  "paidAt": "2025-03-18T12:00:00.000Z",
  "createdAt": "2025-03-18T11:30:00.000Z"
}
```

### 订单列表接口

**接口**: `GET /api/payment/orders`

**查询参数**:
- `type`: 订单类型（subscription/points）
- `status`: 订单状态
- `page`: 页码
- `limit`: 每页数量

## 🔒 八、安全注意事项

### 1. 签名验证
- ✅ 必须验证支付宝回调的签名
- ✅ 使用支付宝公钥验证
- ✅ 签名验证失败立即拒绝

### 2. 金额验证
- ✅ 验证回调金额与订单金额一致
- ✅ 防止金额篡改

### 3. 幂等性处理
- ✅ 同一订单只处理一次
- ✅ 已支付订单直接返回成功

### 4. 数据保护
- ✅ 敏感信息加密存储
- ✅ 使用 HTTPS 传输
- ✅ 日志中脱敏敏感数据

## 🆘 十一、常见问题

### Q1: 回调地址配置后收不到通知？
- 检查回调地址是否可公网访问
- 检查防火墙设置
- 查看服务器日志确认是否收到请求
- 使用支付宝回调测试工具

### Q2: 签名验证失败？
- 检查公钥/私钥是否正确
- 检查密钥格式（去掉 PEM 标记）
- 确认使用的是对应的密钥对

### Q3: 订单状态不更新？
- 检查数据库连接
- 查看订单处理逻辑
- 确认回调返回 "success"

### Q4: 如何申请发票？
- 支付成功后，用户可在个人中心申请
- 后台管理系统导出开票数据
