## 一、背景与目标

### 当前问题

公版书导入后采用**链式结构**（每章 parent_id 指向上一章），无法体现原著的层级关系：

```
当前：第一回 → 第二回 → 第三回 → ...（扁平链式）

期望：
  卷一
    ├── 学而第一
    └── 为政第二
  卷二
    ├── 八佾第三
    └── 里仁第四
```

### 目标

1. **自动识别层级**：根据章节标题自动判断层级关系（卷 → 章 → 节）
2. **折叠式展示**：前端支持折叠/展开卷级目录
3. **兼容分支功能**：用户可以从任意层级（包括卷）创建分支

---

## 二、层级识别规则

### 层级1：卷/部/编（容器级）

| 格式 | 示例 |
|------|------|
| `卷X` | 卷一、卷二、卷首、卷末 |
| `上/中/下卷` | 上卷、中卷、下卷 |
| `第X部` | 第一部、第二部 |
| `上/中/下编` | 上编、中编、下编 |
| `Part X` | Part 1、Part One |

### 层级2：章/回/篇（正文级）

| 格式 | 示例 |
|------|------|
| `第X章` | 第一章、第1章 |
| `第X回` | 第一回、第1回 |
| `XXX第X`（论语格式） | 学而第一、为政第二 |
| `Chapter X` | Chapter 1、Chapter 2 |
| `《XXX》` | 《狂人日记》、《阿Q正传》 |

### 层级3：节/段（细分级）

| 格式 | 示例 |
|------|------|
| `第X节` | 第一节、第1节 |
| `X、` | 一、二、三、 |
| `(X)` | (一)、(二)、(1)、(2) |

### 识别优先级

```
如果标题匹配层级1规则 → level = 1
否则如果标题匹配层级2规则 → level = 2
否则如果标题匹配层级3规则 → level = 3
否则 → level = 2（默认为章节级）
```

## 三、数据结构设计

### JSON 输出格式（text-to-json.ts）

```json
{
  "title": "论语",
  "description": "儒家经典",
  "tags": "儒家,经典,先秦",
  "chapters": [
    {
      "title": "卷一",
      "content": "",
      "level": 1
    },
    {
      "title": "学而第一",
      "content": "子曰：学而时习之...",
      "level": 2,
      "parentTitle": "卷一"
    },
    {
      "title": "为政第二",
      "content": "子曰：为政以德...",
      "level": 2,
      "parentTitle": "卷一"
    },
    {
      "title": "卷二",
      "content": "",
      "level": 1
    },
    {
      "title": "八佾第三",
      "content": "孔子谓季氏...",
      "level": 2,
      "parentTitle": "卷二"
    }
  ]
}
```

### 数据库结构（无需修改）

现有 `nodes` 表已支持层级结构：

```prisma
model nodes {
  id         Int      @id @default(autoincrement())
  story_id   Int
  parent_id  Int?     // 父节点ID（用于构建树）
  title      String
  content    String   // 容器节点可为空
  path       String   // "1", "1.1", "1.2", "2", "2.1"
  ...
}
```

### 导入后的数据示例

| id | parent_id | title | content | path | level |
|----|-----------|-------|---------|------|-------|
| 1 | NULL | 卷一 | (空或序言) | 1 | 1 |
| 2 | 1 | 学而第一 | 子曰：学而... | 1.1 | 2 |
| 3 | 1 | 为政第二 | 子曰：为政... | 1.2 | 2 |
| 4 | NULL | 卷二 | (空) | 2 | 1 |
| 5 | 4 | 八佾第三 | 孔子谓季氏... | 2.1 | 2 |

## 四、前端展示设计

### 折叠式章节列表

```
┌─────────────────────────────────────────┐
│  📚 论语                    [展开全部]    │
├─────────────────────────────────────────┤
│  ▼ 卷一                     [12章]       │
│     ├── 学而第一            1,234字      │
│     ├── 为政第二            2,345字      │
│     ├── 里仁第四            1,567字      │
│     └── 公冶长第五          2,123字      │
│  ▶ 卷二                     [10章]       │
│  ▶ 卷三                     [14章]       │
│  ▶ 卷四                     [11章]       │
│  ...                                     │
└─────────────────────────────────────────┘
```

### 交互行为

| 操作 | 行为 |
|------|------|
| 点击卷标题 | 展开/折叠子章节 |
| 点击章节标题 | 跳转到章节阅读页 |
| 点击卷的"续写"按钮 | 在该卷下创建新章节 |
| 点击章节的"续写"按钮 | 在该章节后创建分支 |

## 六、兼容性考虑

### 向后兼容

- 现有链式结构的公版书**无需迁移**，继续正常工作
- 新导入的书籍自动识别层级
- 前端根据 path 深度判断是否显示折叠控件

### 分支功能

- 用户可以从任意节点（包括卷）创建分支
- 从卷创建分支 = 在该卷下新增一章
- 从章节创建分支 = 续写/创建平行分支
