**核心功能：**

  - 验证今天是否已签到

  - 需要登录

  - 需要登录

  - 返回当月签到记录
  - 返回累计签到统计（总签到数、总积分、补签次数）

- ✅ **签到排行榜** (`GET /api/checkin/leaderboard`)
  - 公开接口

## 📊 签到奖励机制

### 每日签到奖励

签到积分根据连续天数递增：

| 连续天数 | 每日奖励 | 说明 |
|---------|---------|------|
| 1-7天 | 10-22积分 | 每天递增2积分 |
| 8-30天 | 22-37积分 | 每7天递增5积分 |
| 30天以上 | 37+积分 | 每30天递增10积分 |

**计算公式：**
- 1-7天：`10 + (天数-1) * 2`
- 8-30天：`22 + floor((天数-7)/7) * 5`
- 30天以上：`37 + floor((天数-30)/30) * 10`

### 连续签到里程碑奖励

| 连续天数 | 额外奖励 | 特殊权益 |
|---------|---------|---------|
| 7天 | 50积分 | - |
| 30天 | 200积分 | - |
| 100天 | 1000积分 | - |
| 365天 | 5000积分 | - |

### 补签机会获得

- **连续签到奖励**：每连续签到7天，获得1次补签机会
- **码字奖励**：每码1000字，获得1次补签机会（已在milestone-checker中实现）

### 补签规则

- 只能补签最近7天内的日期
- 每次补签消耗1次补签机会
- 补签也获得相应的积分奖励
- 补签不计入连续签到天数

## 📝 API接口文档

### 用户接口（需要登录）

#### 每日签到

```http
POST /api/checkin/daily
Authorization: Bearer <token>
```

**响应示例：**

```json
{
  "success": true,
  "message": "签到成功！",
  "data": {
    "pointsEarned": 12,
    "bonusPoints": 0,
    "bonusMakeupChances": 0,
    "consecutiveDays": 3,
    "totalPoints": 1012,
    "milestoneMessage": ""
  }
}
```

#### 补签

```http
POST /api/checkin/makeup
Authorization: Bearer <token>
Content-Type: application/json

{
  "date": "2026-03-15"
}
```

**响应示例：**

```json
{
  "success": true,
  "message": "补签成功！",
  "data": {
    "pointsEarned": 10,
    "remainingMakeupChances": 2
  }
}
```

#### 获取签到状态

```http
GET /api/checkin/status
Authorization: Bearer <token>
```

**响应示例：**

```json
{
  "consecutiveDays": 5,
  "lastCheckinDate": "2026-03-16T10:00:00Z",
  "makeupChances": 3,
  "canCheckin": false,
  "isMissed": false,
  "nextReward": 20,
  "reason": "今天已经签到过了"
}
```

#### 获取签到历史

```http

```

**响应示例：**

```json
  "records": [

      "id": 1,
      "user_id": 1,

      "is_makeup": false

    "totalCheckins": 50,
```


## 🚀 部署和测试

### 1. 数据库迁移

签到相关表已在schema.prisma中定义，无需额外迁移：
- `checkin_records` - 签到记录表
- `users.consecutive_days` - 连续签到天数
- `users.last_checkin_date` - 最后签到日期
- `users.makeup_chances` - 补签机会

### 2. 测试流程

#### 每日签到测试

1. 登录用户账号
2. 访问个人中心 `/profile`
3. 点击"今日签到"按钮
4. 检查是否获得积分奖励
5. 确认按钮变为"已签到"状态
6. 日历上今天的日期应显示为已签到

#### 连续签到测试

1. 连续多天签到
2. 检查连续天数是否正确增加
3. 检查奖励积分是否递增
4. 连续签到7天时，检查是否获得补签机会

#### 补签测试

1. 确保有补签机会
2. 点击日历上可补签的日期（橙色虚线边框）
3. 确认补签对话框
4. 检查是否消耗补签机会
5. 检查是否获得积分
6. 日历上该日期应标记为补签（橙色）

#### 签到中断测试

1. 连续签到几天
2. 停止签到1天以上
3. 再次签到
4. 检查连续天数是否重置为1
5. 检查是否显示"连续签到已中断"提示
