# AI功能使用指南

## 🚀 快速开始

### 1. 启动服务
```bash
# 启动后端服务
cd api
npm run dev

# 访问前端
open http://localhost:3001
```

### 2. 使用AI续写

#### 方式1: 写作页面直接使用
1. 登录系统
2. 创建新故事
3. 点击"开始写作"
4. 写入至少50字内容
5. 点击"AI续写建议"按钮
6. 选择喜欢的续写方案

#### 方式2: API直接调用
```bash
curl -X POST http://localhost:3001/api/ai/generate \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "storyId": 1,
    "context": "你的故事内容...",
    "style": "悬疑",
    "count": 3
  }'
```

## 🎨 可用的AI风格

### 1. 悬疑 🔍
**特点**: 紧张刺激，充满悬念  
**适合**: 推理、惊悚、犯罪故事  
**示例**: "突然，一阵冷风吹过..."

### 2. 温情 ❤️
**特点**: 温暖治愈，情感细腻  
**适合**: 亲情、友情、成长故事  
**示例**: "她的眼眶突然红了..."

### 3. 科幻 🚀
**特点**: 未来感强，科技元素  
**适合**: 太空、AI、时间旅行  
**示例**: "量子通讯器闪烁着蓝光..."

### 4. 武侠 ⚔️
**特点**: 江湖侠义，武打精彩  
**适合**: 武侠、古装、江湖  
**示例**: "剑气纵横三万里..."

### 5. 现实 🏙️
**特点**: 贴近生活，心理细腻  
**适合**: 都市、职场、情感  
**示例**: "地铁里的人群如潮水般涌动..."

### 6. 浪漫 💕
**特点**: 浪漫唯美，情感丰富  
**适合**: 爱情、青春、校园  
**示例**: "他的眼神里藏着整个星河..."

### 7. 奇幻 🔮
**特点**: 魔幻世界，想象丰富  
**适合**: 魔法、异世界、冒险  
**示例**: "魔法阵在脚下缓缓展开..."

### 8. 脑洞 💡
**特点**: 创意独特，出人意料  
**适合**: 轻松、搞笑、创意  
**示例**: "我醒来发现自己变成了一只猫..."

## 💡 使用技巧

### 提供更好的上下文
```javascript
// ❌ 不好的上下文
"他走了。"

// ✅ 好的上下文
"雨越下越大，他站在门口犹豫了很久，最终还是转身离开了。
她透过窗户看着他的背影消失在雨中，心里五味杂陈。"
```

### 选择合适的风格
- **故事开头** → 使用"脑洞"或"悬疑"吸引读者
- **情感高潮** → 使用"温情"或"浪漫"深化情感
- **剧情转折** → 使用"悬疑"或"脑洞"制造惊喜
- **结尾收束** → 使用"温情"或"现实"升华主题

### 多次生成对比
点击"重新生成"按钮，AI会给出不同的续写方案，选择最满意的一个。

## 📊 AI使用统计

### 查看个人使用情况
```sql
-- 在数据库中查询
SELECT 
  COUNT(*) as total_requests,
  SUM(total_tokens) as total_tokens,
  SUM(cost_usd) as total_cost,
  AVG(response_time_ms) as avg_time
FROM ai_usage_logs
WHERE user_id = YOUR_USER_ID;
```

### 查看最近的AI使用
```sql
SELECT 
  created_at,
  model_name,
  total_tokens,
  cost_usd,
  response_time_ms,
  success
FROM ai_usage_logs
WHERE user_id = YOUR_USER_ID
ORDER BY created_at DESC
LIMIT 10;
```

## 🐛 常见问题

### Q1: 点击AI续写没有反应？
**A**: 检查以下几点：
1. 是否已登录
2. 内容是否至少50字
3. 浏览器控制台是否有错误
4. 后端服务是否正常运行

### Q2: AI生成的内容不理想？
**A**: 尝试：
1. 提供更详细的上下文
2. 切换不同的风格
3. 点击"重新生成"
4. 手动编辑AI生成的内容

### Q3: 成本如何计算？
**A**: 
- 输入Token: $0.25 / 1M tokens
- 输出Token: $1.25 / 1M tokens
- 平均每次生成: $0.0005 - $0.001

### Q4: 如何提高生成质量？
**A**:
1. 提供清晰的故事背景
2. 保持人物设定一致
3. 给出明确的情节方向
4. 使用适合的写作风格

## 🔧 故障排除

### 错误: "nodeId or storyId is required"
**原因**: API参数缺失  
**解决**: 确保传入storyId和context

### 错误: "Not authenticated"
**原因**: 未登录或Token过期  
**解决**: 重新登录获取新Token

### 错误: "AI API Error"
**原因**: Claude API调用失败  
**解决**: 
1. 检查API Key是否正确
2. 检查网络连接
3. 查看后端日志

### 错误: 响应时间过长
**原因**: 
1. 上下文过长（>2000字）
2. 网络延迟
3. API服务繁忙

**解决**:
1. 精简上下文内容
2. 稍后重试
3. 考虑使用更快的模型

## 📈 性能优化建议

### 1. 控制上下文长度
```javascript
// 自动截断过长内容
context.substring(0, 2000)
```

### 2. 使用缓存
相似的请求可以缓存结果，减少API调用

### 3. 批量生成
一次生成多个选项，让用户选择

### 4. 异步处理
使用WebSocket或长轮询处理长时间请求
