# StoryTree 互动功能与发现页榜单总结

> 更新时间：2026-04-21（已移除整本故事收藏功能，同步更新榜单排序依据）

---

## 一、所有互动功能梳理

### 1. 章节打分（评分）

| 项目 | 内容 |
|------|------|
| 接口 | `POST /api/nodes/:id/rate` |
| 参数 | `score: 1–5` |
| 逻辑 | upsert `ratings` 表（每人每章只能打一次，重复打分会覆盖），实时重新计算并更新 `nodes.rating_avg` 和 `nodes.rating_count` |
| 积分 | ❌ 打分行为本身**不触发积分**（只有被评论、被收藏章节、被追更才给作者积分） |

---

### 2. 评论点赞（投票）

| 项目 | 内容 |
|------|------|
| 接口 | `POST /api/comments/:commentId/vote` |
| 参数 | `voteType: like / dislike` |
| 逻辑 | 对评论投票；相同 `voteType` 再次调用会自动切换（取消 / 重新投票） |
| 积分 | ❌ 不触发积分 |

---

### 3. 章节收藏

| 项目 | 内容 |
|------|------|
| 接口 | `POST /api/bookmarks/node/:nodeId`（切换型：收藏 / 取消收藏） |
| 逻辑 | 写入 `node_bookmarks` 表；首次收藏时给章节作者发放积分（防刷：检查历史 `point_transactions` 记录）；同时发通知给作者 |
| 积分 | ✅ 章节作者获得 **+2 积分**（仅首次） |

---

### 4. 追更故事（关注故事）

| 项目 | 内容 |
|------|------|
| 接口 | `POST /api/stories/:id/follow` / `DELETE /api/stories/:id/follow` |
| 逻辑 | 写入 `story_followers` 表；首次追更给故事作者发放积分；发通知给作者 |
| 积分 | ✅ 故事作者获得 **+5 积分**（仅首次） |

---

### 5. 关注用户（人与人之间）

| 项目 | 内容 |
|------|------|
| 接口 | `POST /api/users/:id/follow` / `DELETE /api/users/:id/follow` |
| 逻辑 | 写入 `follows` 表；被关注用户收到通知；不能关注自己 |
| 积分 | ❌ 不触发积分 |

---

### 6. 发表评论

| 项目 | 内容 |
|------|------|
| 接口 | `POST /api/nodes/:node_id/comments`（支持 `parent_id` 嵌套回复） |
| 逻辑 | 评论字数需 ≥ 5 字；首次评论给章节作者发放积分；发通知给章节作者 |
| 积分 | ✅ 章节作者获得 **+2 积分**（首次） |

---

### 7. 创作章节（码字）

| 项目 | 内容 |
|------|------|
| 接口 | `POST /api/nodes`（创建）、`PUT /api/nodes/:id`（更新）、`POST /api/nodes/:id/publish`（发布草稿） |
| 逻辑 | 按字数发放积分；创作超过特定字数里程碑会解锁**徽章** |
| 积分 | ✅ 作者按字数获得积分（`word_count` 字段动态计算） |

---

## 二、其他互动功能（未在主流程中提及）

| 功能 | 接口 | 说明 |
|------|------|------|
| **打赏（Tip）** | `POST /api/points-features/tips` | 用积分打赏作者；发送方扣积分，接收方加积分；支持指定故事 / 章节 |
| **置顶故事** | `POST /api/points-features/stories/:storyId/pin` | 消耗积分将故事置顶 N 天 |
| **置顶评论** | `POST /api/points-features/comments/:commentId/pin` | 消耗积分置顶评论 |
| **举报章节** | `POST /api/nodes/:id/report` | 可选原因：垃圾广告 / 不当内容 / 暴力血腥 / 色情低俗 / 抄袭侵权 / 其他 |
| **分享记录** | `POST /api/shares` | 记录分享到哪个平台（微信 / 微博 / 复制链接等），登录 / 未登录均可 |
| **每日签到** | `POST /api/points/daily-checkin` | 每日登录签到，获得 **+1 积分** |
| **完善资料** | 用户更新 profile 时触发 | `COMPLETE_PROFILE` +10 积分（规则已定义，触发逻辑待确认） |
| **徽章系统** | `GET /api/badges` | 按累计字数里程碑解锁徽章 |
| **协作申请** | `POST /api/collaboration-requests` | 申请加入他人故事共创，作者审核通过后成为协作者 |
| **邀请码** | `POST /api/invitations/generate` | 生成邀请码，被邀请者注册时双方获得积分 |
| **会员订阅** | `POST /api/membership` | 付费开通会员（微信支付），享受额外权益 |

## 三、积分触发规则汇总

| 行为 | 积分变化 | 受益方 | 备注 |
|------|----------|--------|------|
| 每日签到 | +1 | 本人 | 每天限一次 |
| 创作章节 | 按字数 | 本人 | 码字奖励 |
| 发布首个故事 | +50 | 本人 | 一次性 |
| 完善个人资料 | +10 | 本人 | 规则已定义 |
| 被他人评论章节 | +2 | 章节作者 | 首次 |
| 被他人收藏章节 | +2 | 章节作者 | 首次，防刷 |
| 被他人追更故事 | +5 | 故事作者 | 首次 |
| 被他人打赏 | 打赏金额 | 故事/章节作者 | 实时到账 |
| 打赏他人 | -打赏金额 | 发送方 | 实时扣除 |
| 置顶故事 | 消耗积分 | 发起方 | 按天收费 |
| 置顶评论 | 消耗积分 | 发起方 | — |

## 四、发现页榜单排序依据

发现页有 **3 个榜单 Tab**，前端传 `sort` 参数给 `GET /api/stories`，后端排序逻辑如下：

| 榜单 | `sort` 值 | 排序字段（`orderBy`） |
|------|-----------|----------------------|
| 🔥 热门榜 | `popular` | 追更数（`followers._count`）降序 → 章节数（`nodes._count`）降序 → 创建时间降序 |
| 📈 趋势榜 | `trending` | 章节数（`nodes._count`）降序 → 追更数（`followers._count`）降序 → 创建时间降序 |
| 🆕 最新榜 | `latest` | 创建时间（`created_at`）降序 |

**榜单卡片展示字段**：章节数（`_count.nodes`）、追更数（`_count.followers`）

**分类标签筛选**（与榜单正交组合，横向滚动）：
奇幻、科幻、悬疑、言情、武侠、历史、都市、恐怖、喜剧、冒险、推理、古风、末世、穿越、校园、职场
