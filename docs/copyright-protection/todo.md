# 第二部分：版权与合规功能

## 六、创作协议相关功能

#### 1.1 主创作者不活跃处理机制

**需求来源**：`creation-agreement.md` 第四条 4.1

**功能描述**：
- 若主创作者连续 180 天未登录平台，且故事有活跃续写社区：
  - 平台将指定最活跃的贡献者作为"故事管理员"
  - 故事管理员享有主创作者的部分管理权利（不包括收益权转移）
  - 主创作者重新活跃后，可申请恢复管理权

**实现状态**：❌ 未实现

**技术要点**：
- [ ] 设计"活跃贡献者"算法（考虑：续写章节数、字数、互动量、最近活跃度等）
- [ ] 设计"活跃续写社区"判定标准（如：最近30天是否有新续写）
- [ ] 实现定时任务：每日检查主创作者活跃度
- [ ] 实现"故事管理员"角色和权限体系
- [ ] 实现权限转移/恢复流程
- [ ] 设计通知机制（主创作者、新任管理员、社区成员）
- [ ] 数据库表设计：管理员变更记录表

**优先级**：P2（中优先级，故事生态成熟后需要）

**实现状态**：❌ 未实现

**技术要点**：
- [ ] 设计"非正史"标记的数据模型（标记状态、标记原因、标记时间等）

- [ ] 实现平台审核流程（标记是否需要平台审核？）
