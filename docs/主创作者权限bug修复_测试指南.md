# 主创作者权限Bug修复 - 测试指南

## 🎯 测试目标

验证主创作者在任何可见性设置下都能查看自己故事的分支图和统计分析。

## 📋 测试前准备

1. **确保API服务正在运行**
   ```bash
   cd /Users/jinkun/storytree/api
   npm run dev
   ```

2. **确保前端服务正在运行**
   - 访问 http://localhost:3001

## 🧪 测试步骤

### 步骤1：登录为主创作者

1. 访问 http://localhost:3001/login
2. 使用主创作者账号登录（例如：用户ID=1的账号）

### 步骤2：测试公开故事（基准测试）

1. 访问 http://localhost:3001/story-settings.html?id=1
2. 设置可见性为"公开"
3. 点击"保存设置"
4. 访问 http://localhost:3001/story.html?id=1
5. **验证**：
   - [ ] 章节列表可见
   - [ ] 分支图可见（点击"分支图"标签）
   - [ ] 统计分析可见（点击"统计分析"标签）

### 步骤3：测试关注者可见故事

1. 访问 http://localhost:3001/story-settings.html?id=1
2. 设置可见性为"关注者可见 - 仅关注我的用户可见"
3. 点击"保存设置"
4. 访问 http://localhost:3001/story.html?id=1
5. **验证**：
   - [ ] 章节列表可见
   - [ ] 分支图可见（点击"分支图"标签）✅ **修复后应该可见**
   - [ ] 统计分析可见（点击"统计分析"标签）✅ **修复后应该可见**

### 步骤4：测试私密故事

1. 访问 http://localhost:3001/story-settings.html?id=1
2. 设置可见性为"私密 - 仅我和协作者可见"
3. 点击"保存设置"
4. 访问 http://localhost:3001/story.html?id=1
5. **验证**：
   - [ ] 章节列表可见
   - [ ] 分支图可见（点击"分支图"标签）✅ **修复后应该可见**
   - [ ] 统计分析可见（点击"统计分析"标签）✅ **修复后应该可见**

## 🔍 调试方法

如果仍然无法看到分支图和统计分析：

### 1. 打开浏览器开发者工具

- Chrome/Edge: 按 `F12` 或 `Cmd+Option+I` (Mac)
- Firefox: 按 `F12` 或 `Cmd+Option+I` (Mac)

### 2. 查看Console标签

查找以下日志：
- ✅ "树状图数据加载成功:" - 表示加载成功
- ❌ "加载树状图失败: 403" - 表示权限被拒绝（bug未修复）
- ❌ "加载树状图失败: 401" - 表示未登录或token失效

### 3. 查看Network标签

1. 刷新页面
2. 找到 `/api/stories/1/tree` 请求
3. 检查：
   - **Request Headers**: 应该包含 `Authorization: Bearer <token>`
   - **Status Code**: 应该是 `200` (成功)
   - **Response**: 应该包含 `tree` 和 `stats` 数据

### 4. 常见问题排查

#### 问题1: 403 Forbidden
**原因**: 后端权限检查失败  
**解决**: 确认后端代码已更新并重启服务

#### 问题2: 401 Unauthorized
**原因**: token无效或已过期  
**解决**: 重新登录

#### 问题3: Network Error
**原因**: API服务未运行  
**解决**: 启动API服务 `cd api && npm run dev`

#### 问题4: 分支图显示空白
**原因**: 数据加载成功但渲染失败  
**解决**: 
1. 检查Console是否有JavaScript错误
2. 确认故事至少有一个章节
3. 刷新页面重试
