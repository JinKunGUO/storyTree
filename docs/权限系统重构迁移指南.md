## 📊 数据变更说明

### 删除的表
- `friendships` - 好友关系表（已改用`follows`表）

### 修改的表

#### `story_collaborators`
**删除的字段：**
- `role` - 角色字段（不再区分owner/editor/viewer）

**新增的字段：**
- `removed_at` - 移除时间（软删除）
- `removed_by` - 移除人ID

**数据处理：**
- `role='owner'`的记录将被删除（因为owner就是author_id）
- `role='editor'`和`role='viewer'`的记录统一为"协作者"

#### `stories`
**新增的字段：**
- `require_collaborator_review` - 协作者内容是否需要审核（默认false）

### 新增的表

#### `delete_requests`
用于协作者删除有子节点的章节时提交审核：
- `id` - 主键
- `node_id` - 章节ID
- `requester_id` - 请求人ID
- `reason` - 删除原因
- `status` - 状态（pending/approved/rejected）
- `reviewed_by` - 审核人ID
- `reviewed_at` - 审核时间
- `created_at` - 创建时间

## 🔄 API变更说明

### 移除的API
无（好友相关API如果存在，应该移除）

### 修改的API

#### POST /api/stories/:id/collaborators
**变更：**
- 移除`role`参数
- 不能将作者添加为协作者

**旧请求：**
```json
{
  "user_id": 123,
  "role": "editor"
}
```

**新请求：**
```json
{
  "user_id": 123
}
```

#### GET /api/stories/:id/collaborators
**变更：**
- 响应中不再包含`role`字段
- 仅返回未被移除的协作者

**旧响应：**
```json
{
  "collaborators": [
    {
      "id": 1,
      "user": {...},
      "role": "editor"
    }
  ]
}
```

**新响应：**
```json
{
  "collaborators": [
    {
      "id": 1,
      "user": {...}
    }
  ]
}
```

#### GET /api/stories/my
**变更：**
- 移除`role`字段
- 添加`isCollaborator`字段

#### GET /api/stories/shared-with-me
**变更：**
- 移除`my_role`字段

## ⚠️ 潜在问题及解决方案

### 问题1: 数据库被锁定

**错误信息：**
```
Error: SQLite database error
database is locked
```

**解决方案：**
```bash
# 1. 停止所有使用数据库的进程
ps aux | grep node
kill <PID>

# 2. 如果问题持续，删除锁文件
rm api/prisma/dev.db-journal
rm api/prisma/dev.db-wal
rm api/prisma/dev.db-shm

# 3. 重新执行迁移
cd api
npx prisma migrate dev
```

### 问题2: Prisma客户端类型不匹配

**错误信息：**
```
Type 'role' does not exist on type 'story_collaborators'
```

**解决方案：**
```bash
# 重新生成Prisma客户端
cd api
npx prisma generate

# 重启TypeScript服务器（VSCode）
# Cmd+Shift+P -> TypeScript: Restart TS Server
```

### 问题3: 旧数据兼容性

**场景：** 前端代码仍然尝试访问`role`字段

**解决方案：**
1. 清除浏览器缓存
2. 强制刷新页面（Cmd+Shift+R）
3. 检查前端代码是否有遗漏的`role`引用
