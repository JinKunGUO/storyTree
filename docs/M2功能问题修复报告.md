## 问题列表及修复方案

### ✅ 问题1：举报功能401未授权错误

**问题描述：**
- 控制台输出：`Failed to load resource: the server responded with a status of 401 (Unauthorized)`
- 错误信息：`举报失败: Error: Not authenticated`

**根本原因：**
1. 举报API路由缺少`authenticateToken`中间件
2. 前端发送的中文举报原因与后端期望的英文代码不匹配

**修复方案：**

1. **添加认证中间件**（`api/src/routes/nodes.ts:402`）
   ```typescript
   router.post('/:id/report', authenticateToken, async (req, res) => {
   ```

2. **添加原因映射**（`api/src/routes/nodes.ts:411-423`）
   ```typescript
   // 映射前端的中文原因到后端的英文代码
   const reasonMap: { [key: string]: string } = {
     '垃圾广告': 'spam',
     '不当内容': 'illegal',
     '暴力血腥': 'violence',
     '色情低俗': 'porn',
     '抄袭侵权': 'copyright',
     '其他': 'other'
   };
   const mappedReason = reasonMap[reason] || reason;
   ```

3. **使用映射后的原因**（`api/src/routes/nodes.ts:460, 472`）
   ```typescript
   reason: mappedReason,  // 而不是原始的reason
   ```

**测试方法：**
1. 登录系统
2. 进入任意章节阅读页
3. 点击举报按钮
4. 选择举报原因并提交
5. 应该看到"举报成功！感谢您为社区环境做出贡献"

### ✅ 问题3：AI徽章不显示

**问题描述：**
- 章节列表中看不到AI徽章
- 章节阅读页中看不到AI标记

**可能原因分析：**

1. **数据库字段问题**
   - 数据库字段名为`ai_generated`（下划线）
   - 前端代码已正确处理：`chapter.aiGenerated || chapter.ai_generated`

2. **数据问题**
   - 数据库中的章节`ai_generated`字段值为`false`
   - 没有实际的AI生成章节用于测试

3. **API返回数据问题**
   - Prisma默认会返回所有字段，包括`ai_generated`
   - 已确认API代码正确

**修复方案：**

1. **添加调试信息**（`web/story.html:1228-1250`）
   ```javascript
   // 调试：输出章节数据
   console.log('渲染章节列表，章节数量:', chapters.length);
   if (chapters.length > 0) {
       console.log('第一个章节数据:', chapters[0]);
       console.log('AI字段检查:', {
           ai_generated: chapters[0].ai_generated,
           aiGenerated: chapters[0].aiGenerated
       });
   }
   
   // 调试每个章节的AI字段
   const hasAI = chapter.aiGenerated || chapter.ai_generated;
   console.log(`章节 ${index + 1} "${chapter.title}" AI标记:`, hasAI);
   ```

2. **添加调试信息到chapter.html**（`web/chapter.html:1004-1014`）
   ```javascript
   // 调试：输出章节数据
   console.log('渲染章节详情:', chapter);
   console.log('AI字段检查:', {
       ai_generated: chapter.ai_generated,
       aiGenerated: chapter.aiGenerated
   });
   console.log('AI生成标记:', aiGenerated);
   ```

3. **改进字段兼容性**（`web/story.html:1261`）
   ```javascript
   <span><i class="fas fa-eye"></i> ${chapter.readCount || chapter.read_count || chapter.views || 0} 阅读</span>
   ```

4. **添加作者等级字段**（`api/src/routes/stories.ts:158`）
   ```typescript
   author: {
       select: { id: true, username: true, level: true }
   }
   ```

**测试方法：**

**方法1：使用SQL手动设置测试数据**
```sql
-- 查看现有章节
SELECT id, title, ai_generated FROM "Node" ORDER BY created_at DESC LIMIT 5;

-- 将某个章节标记为AI生成
UPDATE "Node" SET ai_generated = true WHERE id = 1;  -- 替换1为实际ID
```

**方法2：使用AI续写功能创建AI章节**
1. 进入故事详情页
2. 点击"AI续写"按钮
3. 生成AI章节
4. AI生成的章节会自动标记`ai_generated = true`

**方法3：使用浏览器控制台检查**
```javascript
// 打开故事详情页，在控制台运行：
// 查看返回的数据
fetch('/api/stories/1')  // 替换1为实际故事ID
    .then(r => r.json())
    .then(data => {
        console.log('章节数据:', data.nodes);
        data.nodes.forEach(n => {
            console.log(`${n.title}: ai_generated=${n.ai_generated}`);
        });
    });
```

## 技术要点

### Prisma字段命名
- 数据库字段：`ai_generated`（snake_case）
- JavaScript对象：`ai_generated`（Prisma默认保持原样）
- 前端兼容：同时支持`aiGenerated`和`ai_generated`

### 认证中间件
- `authenticateToken`：必须登录，否则返回401
- `optionalAuth`：可选登录，不强制要求
- `getUserId(req)`：获取当前用户ID（可能为null）

### 举报原因映射

| 不当内容    | illegal    |
| 暴力血腥    | violence   |
| 色情低俗    | porn       |
| 抄袭侵权    | copyright  |

## 常见问题排查

### Q: 举报功能仍然401错误？
A: 
1. 确认已重启后端服务
2. 检查浏览器localStorage中是否有有效token
3. 尝试重新登录

### Q: AI徽章仍然不显示？
A:
1. 打开浏览器控制台，查看调试输出
2. 确认章节数据中`ai_generated`字段的值
3. 如果为`false`，使用SQL手动设置为`true`
4. 清除浏览器缓存后重新加载

### Q: 等级徽章不显示？
A:
1. 确认用户数据中有`level`和`points`字段
2. 检查localStorage中的用户信息
3. 尝试重新登录以刷新用户数据
