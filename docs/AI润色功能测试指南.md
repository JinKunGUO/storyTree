# AI润色功能测试指南

## 🧪 快速测试步骤

### 测试前准备

1. **启动服务**
   ```bash
   cd api
   npm run dev
   ```

2. **确保已登录**
   - 打开浏览器访问 `http://localhost:3000`
   - 登录账号（如果没有账号，先注册）

---

## 📝 测试场景1：write.html写作页面润色

### 步骤1：进入写作页面
1. 访问 `http://localhost:3000/write?storyId=<你的故事ID>`
2. 如果没有故事，先创建一个故事

### 步骤2：输入测试内容
在编辑器中输入以下测试文本：
```
今天天气很好，我出去散步了。路上看到了很多花，还有很多鸟在唱歌。我觉得心情很愉快。
```

### 步骤3：测试润色功能
1. 点击右下角的"AI润色"按钮
2. 观察是否弹出润色模态框
3. 查看6个风格按钮是否正常显示
4. 等待润色完成（约5-15秒）
5. 观察左右对比面板是否正常显示

### 步骤4：测试风格切换
1. 点击"华丽"风格按钮
2. 观察是否自动重新润色
3. 对比不同风格的润色结果

### 步骤5：应用润色
1. 点击"应用润色"按钮
2. 观察编辑器内容是否被替换
3. 检查字数统计是否更新

### 预期结果
- ✅ 模态框正常弹出
- ✅ 风格选择器可点击，active状态正确
- ✅ 润色结果正常显示，原文和润色后文本对比清晰
- ✅ 应用润色后，编辑器内容正确更新
- ✅ 字数统计正确

---

## 📝 测试场景2：chapter.html章节编辑润色

### 步骤1：进入章节阅读页
1. 访问 `http://localhost:3000/chapter?id=<章节ID>`
2. 确保是章节的作者（否则看不到编辑按钮）

### 步骤2：打开编辑模态框
1. 点击工具栏的"编辑"按钮（铅笔图标）
2. 观察编辑模态框是否弹出

### 步骤3：测试润色功能
1. 在内容输入框中输入测试文本（如果已有内容，可以直接使用）
2. 选择润色风格（如"幽默"）
3. 点击"AI润色"按钮
4. 等待润色完成
5. 观察confirm对话框是否显示字数对比

### 步骤4：确认应用
1. 点击"确定"应用润色
2. 观察输入框内容是否更新
3. 检查字数统计是否更新

### 步骤5：保存修改
1. 点击"保存修改"按钮
2. 观察章节内容是否成功保存

### 预期结果
- ✅ 编辑模态框正常显示润色控件
- ✅ 润色按钮和风格选择器正常工作
- ✅ 润色完成后显示确认对话框
- ✅ 应用润色后内容正确更新
- ✅ 保存后章节内容更新

## 🔧 调试技巧

### 查看控制台日志
打开浏览器开发者工具（F12），查看Console标签页：

```javascript
// 应该看到以下日志
✅ AI润色功能初始化完成
用户点击AI润色按钮
调用AI润色API，风格: elegant
润色API响应状态: 200
润色API返回数据: {taskId: 123, polished: "..."}
显示润色结果
```

### 检查网络请求
在Network标签页中查看：
- 请求URL：`/api/ai/v2/polish`
- 请求方法：POST
- 请求头：包含Authorization
- 请求体：包含content和style
- 响应状态：200
- 响应数据：包含polished字段

### 常见问题排查

**问题1：点击润色按钮没反应**
- 检查控制台是否有错误
- 确认按钮事件是否绑定成功
- 检查是否有JavaScript错误

**问题2：API返回401错误**
- 检查是否已登录
- 检查token是否有效
- 检查Authorization头是否正确

**问题3：润色结果不显示**
- 检查API响应数据格式
- 确认polished字段存在
- 检查DOM元素是否正确更新
