# 移动端响应式编码规范

## 强制规则

### 1. 禁止在 JS 中使用固定宽度内联样式
- 禁止：`style="width: 300px"` / `style="min-width: 200px"`
- 允许：使用 CSS class，或使用相对单位 `width: 100%`、`max-width: Npx`、`flex: 1`
- 如果必须使用固定宽度，必须同时加 `max-width: 100%` 防护

### 2. 所有 display: flex 必须考虑换行
- 在 JS 内联样式中写 `display: flex` 时，必须同时加 `flex-wrap: wrap`
- 唯一例外：父容器中只有 2 个子元素且内容极短（如两个按钮）
- 在 CSS 文件中，如果不加 flex-wrap，必须在对应的 media query 中处理窄屏情况

### 3. 所有 flex 子项必须有 min-width: 0
- flex: 1 的子项必须加 `min-width: 0` 防止内容溢出
- 已通过全局基线 `* { min-width: 0 }` 覆盖（`responsive.css` Section 1b）
- 但 JS 内联样式中设置 `flex: 0 0 auto` 的元素需要注意不要设置过大的固定宽度

### 4. 禁止使用 calc() 产生可能为负的宽度
- 禁止：`width: calc(100% - 160px)` — 当容器 < 160px 时产生负值
- 允许：配合 `min-width: 60px` 或改用 `flex: 1` 布局

### 5. 容器必须有溢出防护
- 所有页面主容器必须有 `overflow-x: hidden` 或 `max-width: 100vw`
- 已通过全局基线覆盖常见容器 class（`responsive.css` Section 1b）

## 测试要求

- 所有涉及布局的改动，必须在 375px 视口宽度下验证无水平溢出
- 可运行 `node scripts/lint-mobile-styles.js` 检查 JS 内联样式风险
- 可运行 `node scripts/test-mobile-overflow.js` 进行自动化溢出检测
